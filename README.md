# 🧪 QA Testing Project – Mall CRM System

This repository contains a comprehensive QA testing effort for a web-based Mall CRM system.  
It includes structured **Test Cases** and **Bug Reports** covering key modules such as Customer Management and Offers.

---

## 🎯 Key Highlights

- 70+ Test Cases covering critical system flows  
- 20+ real-world bugs documented  
- Covers both **functional and UI/UX testing**  
- Includes **negative and edge case scenarios**  
- Structured and professional QA documentation  

---

## 📌 Project Overview

The purpose of this project is to demonstrate:

- Manual testing skills
- Writing structured test cases
- Identifying and documenting bugs
- Understanding UI/UX and business logic issues
- Reporting issues in a clear and professional format

---

## 🧪 Test Cases

The test cases are organized into two main modules:

### 🔹 Customer Management
- Page load & layout validation
- Form validation (email, phone, date of birth)
- Search & pagination behavior
- UI/UX consistency

### 🔹 Offers Module
- Offer creation & editing
- Date & validity logic
- Image upload & cropping
- Filtering functionality
- Status behavior (Active / Inactive)

---

## 🐞 Bug Reports

A total of **21 Bugs** have been identified and documented with the following structure:

- Bug ID  
- Title  
- Steps to Reproduce  
- Expected Result  
- Actual Result  
- Screenshot  
- Severity  
- Priority  
- Assigned Person  
- Status  
- Comments  
- Devices  

### 🔍 Types of Bugs Found

- Validation issues (e.g. phone number accepts invalid input)
- Business logic errors (e.g. offer status handling with dates)
- UI/UX issues (e.g. invisible controls, redundant buttons)
- Functional issues (e.g. export CSV, filtering)
- Security concern (unauthorized page access)
